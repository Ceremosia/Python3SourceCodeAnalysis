1.2.3 类型的类型

﻿# 1.2.3 类型的类型   在 python 中，任何一个东西都是对象，而每个对象都对应着一种类型。比如说 a = int(1)，a 就是 int 类型，值为 1 的的对象。那么，类型对象的类型是什么呢？这个问题听上去很绕口，实际上却非常重要。对于其他的对象，可以通过与其关联的类型对象确定其类型，那么通过什么来确定一个对象是类型对象呢？

  仔细观察 PyTypeObject 会有一个有趣的发现。在 PyTypeObject 定义的最开始， 可以发现 PyObject_VAR_HEAD。这意味着 Python 中的 type 实际上也是一个继承自 PyVarObject 的对象。

[/Include/cpython/object.h]
struct _typeobject {
    PyObject_VAR_HEAD
    const char *tp_name; /* For printing, in format "<module>.<name>" */
    Py_ssize_t tp_basicsize, tp_itemsize; /* For allocation */
    ......
};

  那么 type 类型的对象是什么呢？答案就是 PyType_Type。

[/Objects/tpeobject.c]
PyTypeObject PyType_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0)
    "type",                                     /* tp_name */
    sizeof(PyHeapTypeObject),                   /* tp_basicsize */
    sizeof(PyMemberDef),                        /* tp_itemsize */
    ......
};

  PyType_Type 在 Python 的类型机制中是一个至关重要的对象，所有用户自定义 class 所对应的 PyTypeObject 对象都是通过 这个对象创建的。一般的 PyTpyeObject 和 PyType_ Type 的关系：

>>> class A(object):
...     pass
... 
>>> A.__class__
<class 'type'>
>>> b = 1
>>> b.__class__
<class 'int'>
>>> int.__class__
<class 'type'>
>>> type.__class__
<class 'type'>

  这个—再出现的 <class 'type'> 就是 Python 内部的 PyType_ Type，它是所有 class 的 class, 所以它在 Python 中被称 为 metaclass。

  我们接看来看 PyLong_Type（上面<class 'int'>的对应的类型对象） 是怎么和 PyType_Type 建立关系的。前面提到，在 Python 中，每一个对象都将自己的引用计数、类型信息保存在开始的部分中（即基类 PyObject 或 PyVarObject 中）。

  观察 PyType_Type 开始部分，有个宏 PyVarObject_HEAD_INIT(&PyType_Type, 0)，PyLong_Type 和 PyFloat_Type 也有类似的宏。

[/Objects/longobject.c]
PyTypeObject PyLong_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0)
    "int",                                      /* tp_name */                                                            
    offsetof(PyLongObject, ob_digit),           /* tp_basicsize */
    sizeof(digit),                              /* tp_itemsize */
    ......
};

[/Objects/floatobject.c]
PyTypeObject PyFloat_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0)
    "float",
    sizeof(PyFloatObject),
    ......
};

  这是 Python 为了方便对这部分内存的初始化，提供的几个有用的宏：

[/Include/object.h]
#ifdef Py_TRACE_REFS
/* Define pointers to support a doubly-linked list of all live heap objects. */
#define _PyObject_HEAD_EXTRA            \
    PyObject *_ob_next;           \
    PyObject *_ob_prev;

#define _PyObject_EXTRA_INIT _Py_NULL, _Py_NULL,

#else
#  define _PyObject_HEAD_EXTRA
#  define _PyObject_EXTRA_INIT
#endif

/* PyObject_HEAD defines the initial segment of every PyObject. */
#define PyObject_HEAD                   PyObject ob_base;

#define PyObject_HEAD_INIT(type)        \                                       
    { _PyObject_EXTRA_INIT              \
    1, (type) },

#define PyVarObject_HEAD_INIT(type, size)       \
    { PyObject_HEAD_INIT(type) (size) },

  再回顾一下 PyObject 和 PyVarObect 的定义，初始化的动作就一目了然了。实际上，这些宏在各种内建类型对象的初始化中被大量地使用着。

  在没有定义 Py_TRACE_REFS 时，PyVarObject_HEAD_INIT(&PyType_Type, 0) 会展开为：

{
1, &PyType_Type }, 0 },

  恰好对应 PyVarObject

[/Include/object.h]
// PyObject 定义
struct _object {
    _PyObject_HEAD_EXTRA                                                        
    Py_ssize_t ob_refcnt;
    PyTypeObject *ob_type; 
};
// PyVarObject 定义
typedef struct {
    PyObject ob_base;                                                           
    Py_ssize_t ob_size; /* Number of items in variable part */
} PyVarObject;
